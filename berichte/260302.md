## Übung (ssh auf Metasploitable)      
### Aufgabenstellung:        
Als Vorbereitung für die folgende Übung.        

Konfiguriere passwortloses ssh Login auf Metasploitable mit Hilfe eines ssh Public/Private keys.       

Betrachte besonders die notwendigen Änderungen in ~/.ssh/authorized_keys, diese Datei muss in der nachfolgenden Übung manipuliert werden.           

Hinweis: der ssh-server auf Metasploitable unterstützt nur Schlüssel im RSA Format (nicht die neueren default Formate wie ed255…).
Der Kali ssh Client lehnt die Verbindung mit diesem alten Schlüssel standardmäßig ab, aber man kann es mit Optionen wieder erzwingen.
Bei ssh Verbindungsproblemen hilft häufig die Verwendung der Verbose Optionen des ssh clients (-v, -vv oder -vvv).

### Lösung:     
### RSA - Key auf Kali erzeugen      
Befehl: ```$ ssh-keygen -t rsa -b 2048 -m PEM -f ~/.ssh/metasploitable_rsa ```       
![](https://github.com/user-attachments/assets/cae98428-14cd-4564-9391-5b432406496d)      
Erklärung: noch auf ChaptGpt schaun!!!!
 
### Public Key anzeigen lassen      
Befehl: ```$ cat ~/.ssh/metasploitable_rsa.pub```       
![](https://github.com/user-attachments/assets/6db15bb3-bcf8-49bf-99d4-09ff70a91275)   

### per Passwort von Kali auf Metasploitable einloggen:      
Befehl: ```$ ssh msfadmin@192.168.18.51```     
Info: dieser Befehl funktioniert nicht, weil der SSH- Server alte Algorithmen benutzt.      
-> Fehlerbehebung:      
Befehl:      
```$ ssh -o HostKeyAlgorithms=+ssh-rsa \
-o PubkeyAcceptedAlgorithms=+ssh-rsa \
msfadmin@192.168.18.51```
        
![](https://github.com/user-attachments/assets/d8a9813a-20e0-4bb4-97d6-0d997ead9086)        
- Auf dem Foto kann man erkennen, damm man nur im Kali auf Metasploitable eingeloggt ist.       

### ssh Verzeichnis im Metasploitable anlegen
Befehle:
```mkdir -p ~/.ssh
chmod 700 ~/.ssh```

### authorized_keys erstellen:     
```nano ~/.ssh/authorized_keys```

-> key an diese Datei anhängen:       
Befehle:   
```$ cat ~/.ssh/metasploitable_rsa.pub | ssh \
-o KexAlgorithms=+diffie-hellman-group1-sha1 \
-o HostKeyAlgorithms=+ssh-rsa \
msfadmin@192.168.18.51 "cat >> ~/.ssh/authorized_keys" ```
