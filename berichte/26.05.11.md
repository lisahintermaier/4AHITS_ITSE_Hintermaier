# Linux Permissions        
## Angabe: Vorbereitung der Übungsumgebung          
Betrachte folgendes Skript und finde heraus was es macht. Anschließend führe das Skript als User kali mit sudo aus, um die Laborumgebung einzurichten:                  
```sh
#!/bin/bash

# Gruppen anlegen
groupadd music
groupadd wrestle
groupadd emperors

# User anlegen und Gruppen zuweisen
useradd -m -G music elvis
useradd -m -G music,wrestle blondie
useradd -m -G wrestle hogan
useradd -m -G emperors nero

# Passwörter auf 'kali' setzen
echo "elvis:kali" | chpasswd
echo "blondie:kali" | chpasswd
echo "hogan:kali" | chpasswd
echo "nero:kali" | chpasswd

# Lab-Struktur vorbereiten
mkdir -p /tmp/lab/gigs /tmp/lab/drafts
echo "Draft" > /tmp/lab/drafts/lyrics.txt
echo "Plans" > /tmp/lab/wrestle_plans.txt
chown -R root:root /tmp/lab
chmod -R 755 /tmp/lab
````
## Lösung: Was macht das Skript  
Das Skript erstellt zuerst die Gruppen music, wrestle und emperors.         
Danach werden vier Benutzer angelegt und den passenden Gruppen zugewiesen.              
Für alle Benutzer wird das Passwort auf kali gesetzt.                   
Anschließend erstellt das Skript eine kleine Ordnerstruktur unter /tmp/lab sowie einige Testdateien.                            
Zum Schluss werden alle Dateien und Ordner dem Benutzer root zugewiesen und mit den Rechten 755 versehen, sodass normale Benutzer die Dateien lesen, aber nicht verändern können.
