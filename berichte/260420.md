## Aufabenstellung
### Übung (netcat banner grabbing II)
Erweitere das Kommando aus der vorhergehenden Übung so, dass die HTTP-Anfrage (Request) nicht über stdin eingegeben werden muss sondern per pipe | an nc übergeben wird.

Hinweis:

Die leere Zeile ist im HTTP Protokoll als die Zeichenfolge CRLF definiert.

CR (Carriage Return): Steuerzeichen mit ASCII-Wert dezimal 13 oder hexadezimal 0x0D – \r in Programmen.
LF (Line Feed): Steuerzeichen mit ASCII-Wert dezimal 10 oder hexadezimal 0x0A – \n in Programmen.
Verwende das Tool printf (statt echo) um \r\n verwenden zu können.

### Lösung: 
Der erste Versuch mit der HTTP Anfrage auf die Website wurde wieder von der Firewall unterbrochen:           
![](https://github.com/user-attachments/assets/f47efeca-f9da-479c-9d0a-8fa29fe63cf6)       
 
Weitere Versuch mittels  ungülitgen Request:      
![](https://github.com/user-attachments/assets/2087ada3-e7ea-4699-b033-6751ed5c0f3e)         
-> hier sieht man wieder den Apache server      
-> Code Zeilen erklären:    
`$ printf "HEAD / HTTP/1.0\r\n\r\n" | nc example.com 80`              
   - HEAD fragt nur die Header der Startseite (/) ab, ohne den eigentlichen Inhalt.  
   - \r\n\r\n beendet korrekt die Anfrage.
   - Das Pipe-Zeichen | leitet diese Anfrage an nc example.com 80 weiter.
   - nc (Netcat) baut eine TCP-Verbindung zu example.com auf Port 80 (HTTP) auf und sendet die Anfrage, woraufhin der Server mit seinen Headern antwortet.
